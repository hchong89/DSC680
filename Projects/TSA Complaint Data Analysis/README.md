# TSA Complaint Data Analysis

## Overview
This project analyzes Transportation Security Administration (TSA) complaint data to identify trends in passenger complaints across U.S. airports and states. The project focuses on understanding customer experience issues, complaint categories, and geographic patterns using data visualization and exploratory data analysis techniques.

The analysis combines multiple datasets related to airport complaints and location information to generate insights into airport service trends and passenger concerns.

---

## Technologies Used
- Python
- Pandas
- Matplotlib
- Seaborn
- Plotly
- Jupyter Notebook

---

## Data Sources
The project combines multiple datasets including:
- TSA complaint records
- Complaint category information
- Airport location and code datasets

The datasets were cleaned, merged, and transformed to support analysis and visualization.

---

## Project Objectives
- Analyze TSA complaint trends across airports
- Identify the most common complaint categories
- Explore geographic patterns in complaints
- Visualize complaint distributions across states
- Understand customer experience trends in air travel

---

## Data Preparation
The project included:
- Cleaning missing values
- Standardizing airport codes
- Merging multiple datasets
- Aggregating complaint counts
- Formatting date variables
- Preparing geographic mapping data

---

## Exploratory Data Analysis
Several visualization techniques were used to analyze the data, including:
- Heatmaps
- Choropleth maps
- Bar charts
- Box plots
- Time trend analysis

The analysis focused on identifying high-volume complaint locations and recurring passenger concerns.

---

## Key Findings
- Certain airports consistently reported higher complaint volumes
- Complaint trends varied across states and airport regions
- Customer service and screening-related issues appeared frequently across the dataset
- Geographic visualizations highlighted regional concentration patterns
